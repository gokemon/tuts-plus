                        Bruno Ferreira 24 days ago

                        In your example for the user schema, you add a pre hook to set up a creation date. IMHO, it is better using SchemaDefinition:

                        export let userSchema: Schema = new Schema({
    createdAt: {
        type: Date,
        required: true,
        default: Date.now
    },
    email: {
        type: String,
        required: true
    },
    firstName: {
        type: String,
        required: true
    },
    lastName: {
        type: String,
        required: true
    }
});

                        With this, you can set fields as required (like I did), and also set defaults and drop the pre hook. Of course, when more complex logic is needed, the pre hook is mandatory :)

                                                    Brian Love Mod Idan 3 months ago

                                                    Idan good question.

                                                    You need to do both: define the custom method signatures in your model's interface, IUserModel in this case, as well as implementing the methods when defining your schema.

                                                    The Schema class has two properties: methods and statics. Each of these is an object of currently defined methods or static functions that are part of the schema.

                                                    In my example you would define a new property in the methods Object property in the userSchema object. Here is what that might look like.

                                                    In models/user.ts:

                                                    export interface IUserModel extends IUser, Document {
  isInGroup?(group: IGroupModel): boolean;
}


                                                    In schemas/user.ts:

                                                    userSchema.methods.isInGroup = function(group: IGroupModel): boolean {
                                                      for (let id of this.groups) {
                                                        if (group._id.toString() === id.toString()) {
                                                          return true;
                                                        }
                                                      }
                                                      return false;
                                                    };

                                                            Brian Love Mod Jordan 3 months ago

                                                            Jordan good question. I might do a new post on adding instance methods and static functions to mongoose schemas using TypeScript. But, let me also comment here.

                                                            First, we need to define a new interface. I'll add it to the models/user.ts file and name it IUserModelStatic. It will extend Model. Here is what that might look like:

                                                            export interface IUserModelStatic extends Model<IUserModel>  {
  findByLogin(email: string, password: string): DocumentQuery<IUserModel[], IUserModel>;
}

                                                            Next, we need to implement this on the statics property of our Schema object:

                                                            userSchema.statics.findByLogin = function(email: string, password: string): DocumentQuery<IUserModel[], IUserModel> {
                                                              let conditions = {
                                                                email: email,
                                                                password: encrypt(password)
                                                              };
                                                              return this.find(conditions);
                                                            };

                                                            Finally, when creating your models via mongoose's model() function you can specify the generic value for the static class:

                                                            let user = connection.model<IUserModel, IUserModelStatic>("User", userSchema);

                                                                          Brian Love Mod Idan 3 months ago

                                                                          I'm not sure if I understand, which may be due to my limited knowledge of mongoose and MongoDB - I'm still learning. Let me try to answer your question to the best of my ability.

                                                                          Schema is a class that is defined by mongoose. We create a new instance of the Schema class, and then provide this when creating a new model (using the mongoose.model() method.

                                                                          Here is what the docs say about a schema:

                                                                          Schemas not only define the structure of your document and casting of properties, they also define document instance methods, static Model methods, compound indexes and document lifecycle hooks called middleware.

                                                                          I was showing how to create document instance methods (using the methods property) as well as static Model methods (using the statics property).

                                                                          In terms of architecture and design, that is entirely dependent on your preferences and needs. In general, I think it makes sense to take advantage of document instance methods and static Model methods for common tasks. But how common a task is will depend on your application's needs. In my example I was showing how you might using a document instance method to check if a user is in a group using user.isInGroup(group). This is something my application needs to do a lot, so I find it helpful to have this implemented on the document instance.

                                                                          I'm not sure if I answered your question, but I hope this is helpful. If not, maybe share a code sample with us or create a plunker.

                                                                                      Brian Love Mod Jordan 3 months ago

                                                                                      Jordan glad you got it working.

                                                                                      I want to preface my answer saying that I am constantly learning, and while I do have a solution for your question, it may not be the *best* solution.

                                                                                      The Server class has a private property named model. The model object stores all of the models for my application. When building your routes, add an additional argument to the static create() method to accept the model. Here are the appropriate snippets in my server.ts file:

                                                                                      export class Server {

                                                                                        private model: IModel;

                                                                                        constructor() {
                                                                                          this.model = Object();
                                                                                        }

                                                                                        public config() {
                                                                                          //use q promises
                                                                                          global.Promise = require("q").Promise;
                                                                                          mongoose.Promise = global.Promise;

                                                                                          //connect to mongoose
                                                                                          let connection: mongoose.Connection = mongoose.createConnection(MONGODB_CONNECTION);

                                                                                          //create models
                                                                                          this.model.user = connection.model<iusermodel>("User", userSchema);
                                                                                        }

                                                                                        private routes() {
                                                                                          let router: express.Router;
                                                                                          router = express.Router();

                                                                                          //IndexRoute
                                                                                          IndexRoute.create(router, this.model);
                                                                                        }
                                                                                      }

                                                                                      And then the IndexRoute is updated to:

                                                                                      export class IndexRoute extends BaseRoute {

                                                                                        public static create(router: Router, model: IModel) {
                                                                                          //log
                                                                                          console.log("[IndexRoute::create] Creating index route.");

                                                                                          //add home page route
                                                                                          router.get("/", (req: Request, res: Response, next: NextFunction) => {
                                                                                            new IndexRoute(model).index(req, res, next);
                                                                                          });
                                                                                        }

                                                                                        public index(req: Request, res: Response, next: NextFunction) {
                                                                                          this.render(req, res, "app/index");
                                                                                        }
                                                                                      }

                                                                                      The short answer to your question is: you need to pass the model into your routes. I imagine there is a better approach to this using a dependency injection framework.

                                                                                        Brian Love Mod viz 3 months ago

                                                                                        viz thank you for the kind words.

                                                                                        Yes, I agree it is duplicative to have the model specification in both the src/interfaces files and in the src/schemas files. The reason for this is because the schemas are not exactly the same as the interfaces. For example, let's say I have two collections in MongoDB:

                                                                                        1. users
                                                                                        2. messages

                                                                                        A message is associated with a user. The message schema might look like:

                                                                                        import { Schema } from "mongoose";

                                                                                        export var messageSchema: Schema = new Schema({
                                                                                          author: { type: Schema.Types.ObjectId, ref: "User" },
                                                                                          createdAt: Date,
                                                                                          body: String
                                                                                        });

                                                                                        In the example code above we have defined that the author property is an ObjectId and references a user. This would not work in the interface specification.

                                                                                        To take the example a bit further, the user interface might look like:

                                                                                        import { ObjectID } from "mongodb";
                                                                                        import { IGroupModel } from "../models/group";

                                                                                        export interface IUser {
                                                                                          email?: string;
                                                                                          firstName?: string;
                                                                                          lastName?: string;
                                                                                          groups?: Array<igroupmodel |="" string="" |="" objectid="">;
                                                                                          password?: string;
                                                                                        }

                                                                                        In the IUser interface I have defined that the groups property can be an array of group model objects (using population in mongoose), an array of strings or an array of ObjectIDs.

                                                                                        So, while the schema appears to duplicate the interfaces they are indeed different when you start working with populations.

                                                                                        Again, please do let me know if there is a better way. I'm always eager to learn.
